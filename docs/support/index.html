What is ILS?

ILS (Intelligent Language System) is a native iOS and macOS client for Claude Code, Anthropic's official CLI for Claude. It provides a beautiful, mobile-first interface to interact with your AI coding assistant directly from your iPhone, iPad, or Mac.

Key Features:

  • Real-time chat with Claude Code via Server-Sent Events (SSE)
  • Session management with fork/branch support
  • Project and skill browsing
  • MCP server and plugin management
  • System monitoring (CPU, memory, disk, network)
  • Optional remote access via Cloudflare Tunnel

Getting Started

Prerequisites

  • macOS device with Claude Code installed
  • Xcode 15+ (for building the app)
  • iOS 17+ or macOS 14+ device

Installation

  1. Clone the repository: 
    git clone https://github.com/krzemienski/ils-ios.git
cd ils-ios
  2. Run the setup script to install backend dependencies: 
    ./setup.sh
  3. Open ILSFullStack.xcworkspace in Xcode
  4. Select the ILSApp scheme and your target device
  5. Build and run (⌘R)

Starting the Backend

The backend must be running on your Mac for the iOS/macOS app to connect:

PORT=9999 swift run ILSBackend

The backend will:

  • Start an HTTP server on port 9999
  • Connect to your Claude Code CLI
  • Provide API endpoints for sessions, projects, skills, MCP servers, and plugins

Connecting the App

  1. Launch the ILS app on your iOS/macOS device
  2. On first run, you'll see the server setup sheet
  3. Enter your Mac's local IP address (e.g., 192.168.1.100:9999)
  4. Tap "Connect" - the app will perform a health check
  5. Once connected, you can browse sessions, chat with Claude, and manage projects

FAQ

What is ILS?

ILS is a native iOS and macOS client for Claude Code, allowing you to interact with your AI coding assistant from your mobile device or Mac with a beautiful, responsive interface.

How do I install ILS?

Clone the GitHub repository, run ./setup.sh to install backend dependencies, then open ILSFullStack.xcworkspace in Xcode and build the app.

How do I connect the app to my Mac?

Start the backend on your Mac with PORT=9999 swift run ILSBackend, then launch the iOS/macOS app and enter your Mac's local IP address (e.g., 192.168.1.100:9999) in the server setup sheet.

Can I use ILS remotely (outside my local network)?

Yes! ILS supports Cloudflare Tunnel for secure remote access. Configure a tunnel in Settings → Cloudflare Tunnel, and the app will connect via your tunnel URL.

Is my data secure?

Yes. ILS does not collect any user data. All communication happens between your iOS/macOS device and your self-hosted backend server. See our Privacy Policy for details.

Does ILS require an internet connection?

Only when Claude Code needs to communicate with Anthropic's API. The app itself communicates with your local backend server, which can work offline for local tasks.

Troubleshooting

Backend not running

Error: "Failed to connect to server" or "Health check failed"

Solution: Verify the backend is running on your Mac:

lsof -i :9999 -P -n

You should see a process named ILSBackend listening on port 9999. If not, start the backend:

cd /Users/your-username/Desktop/ils-ios
PORT=9999 swift run ILSBackend

Wrong port

Error: Connection works but no data loads

Solution: Verify the backend is running on port 9999 (not 8080 or 9090). Check the backend logs - the first line should say:

[ NOTICE ] Server starting on http://0.0.0.0:9999

Cannot find backend binary

Error: swift run ILSBackend fails with "no executable product"

Solution: Make sure you're in the correct directory:

cd /Users/your-username/Desktop/ils-ios
ls Package.swift  # should exist

If Package.swift doesn't exist, you're in the wrong directory.

App crashes on launch

Solution: Clean build in Xcode:

  1. Product → Clean Build Folder (⇧⌘K)
  2. Delete DerivedData: rm -rf ~/Library/Developer/Xcode/DerivedData/ILSApp-*
  3. Build and run again (⌘R)

Chat messages not sending

Solution: Check that:

  • Backend is running and connected
  • Claude Code CLI is installed on your Mac (claude --version)
  • You have an active Anthropic API key configured
  • Check backend logs for errors

SSE streaming stuck

Error: "Taking longer than expected..." appears but no response

Solution: This can happen if Claude Code is slow to respond. The backend has a 30-second initial timeout. If it persists:

  • Check your internet connection
  • Verify your Anthropic API key is valid
  • Try sending a simpler message first
  • Check backend logs for errors
